## 📈 关键教训

### 1. 文档指导的局限性

**发现**：
- 修改 references（method-registry.md）→ 失败
- 修改 SKILL.md（两处）→ 失败
- 添加 MANDATORY 指令 → 失败

**原因**：
- Agent 可能使用缓存的指导
- Agent 可能跳过详细章节
- 文档指导不够强制

**启示**：
- 需要代码层面的强制检查
- 需要在执行流程中插入检查点
- 需要可执行的代码模板

### 2. 快速验证的重要性

**错误**：
- 修改后直接进入完整评测（1 小时）
- 没有快速验证关键词
- 浪费了 2 次完整评测

**改进**：
- 修改后先快速测试（grep 关键词）
- 确认生效后再完整评测
- 节省时间和资源

### 3. 知道何时停止

**发现**：
- 迭代 1: -31.4 分
- 迭代 2: -84.3 分
- 越改越差

**决策**：
- 两次失败后停止
- 不再尝试迭代 3
- 承认方法错误

**正确性**：
- 避免浪费更多时间
- 为未来提供清晰的教训
- 知道何时止损
